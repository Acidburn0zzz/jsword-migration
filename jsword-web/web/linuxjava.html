Installing Java under Fedora Core 3

This will need to be done as root. If you don't have root access, ask your system administrator to do this for you

For whatever reason, JSword and Bible Desktop does not work with /usr/bin/java, which is GNU's java. Actually it is a link to /etc/alternatives/java which is a link to a shell script which runs GNU's java. You will need to download and install Sun's Java. As open source, Fedora does not come with Sun's Java, which is a licensed product.
  1. Download Java 1.4 (Java 1.5 does not work yet)
    http://java.sun.com/j2se/1.4.2/download.html
    1. You need the Java Software Development Kit, if you plan to help develop Bible Desktop or the JSword API.
      Click on the Download J2SE SDK for Other Platforms link.
      Agree to the licensing agreement.
      Under Linux Platform, click on the RPM in self-extracting file (...) link.
    2. Otherwise, you only need the Java Runtime Engine.
      Click on the Download J2SE JRE for Other Platforms link.
      Agree to the licensing agreement.
      Under Linux Platform, click on the RPM in self-extracting file (...) link.
      Save it to a location of your choosing.
  2. Unpack the bin file.
    1. Open a terminal.
    2. cd to the directory containing the download file.
    3. execute the file. The command will be something like
      sh j2re-1_4_2_07-linux-i586-rpm.bin
      or
      sh j2sdk-1_4_2_07-linux-i586-rpm.bin
      Note: the 07 is the point release and yours may be different
      Bible Desktop should work with any version of Sun's Java 1.4.
    4. Page through the license agreement (same one you already read) and when prompted, type yes.
    5. This will create an rpm with -rpm.bin replaced with .rpm
  3. Install Java 1.4
    1. Become root with su -
    2. In the same directory as the unpacked rpm, install it with rpm.
      rpm -ihv j2re-1_4_2_07-linux-i586-rpm.bin
      or
      rpm -ihv j2sdk-1_4_2_07-linux-i586-rpm.bin
  4. Fedora uses alternatives to manage multiple versions of java.
      Make the Java you downloaded an alternative.
      This must be done as root.
    1. Add it as an alternative:
      At the command line execute the following command:
      alternatives --install /usr/bin/java java /usr/java/j2re1.4.2_07/bin/java 300
      or
      alternatives --install /usr/bin/java java /usr/java/j2sdk1.4.2_07/bin/java 301
      The number on the end is the priority to give to the alternative.
      The highest numbered version is considered the best one.
    2. Then make this the default.
      alternative --config java
      This will display a numbered list of the java alternatives and let you change the default.
    3. Confirm the change.
      alternative --display java
    4. If you installed the sdk repeat a) - c) for javac
      alternatives --install /usr/bin/javac javac /usr/java/j2sdk1.4.2_07/bin/javac 301
      alternative --config java
      alternative --display java
  5. Make JAVA_HOME and JRE_HOME be defined for everyone.
    Create /etc/profile.d/java.sh and /etc/profile.d/java.csh
    This must be done as root.
    These are instructions for java.sh. java.csh is left as an exercise to the reader.
    1. If you installed the JRE use the following lines:
      export JAVA_HOME=/usr/java/j2re1.4.2_07
      export J2RE_HOME=$JAVA_HOME
      export PATH=$JAVA_HOME/bin:$J2RE_HOME/bin:$PATH
    2. If you installed the SDK use the following lines:
      export JAVA_HOME=/usr/java/j2sdk1.4.2_07
      export J2RE_HOME=$JAVA_HOME/jre
      export PATH=$JAVA_HOME/bin:$J2RE_HOME/bin:$PATH
    3. If you installed both the SDK and the JRE use the following lines:
      export JAVA_HOME=/usr/java/j2sdk1.4.2_07
      export J2RE_HOME=/usr/java/j2re1.4.2_07
      export PATH=$JAVA_HOME/bin:$J2RE_HOME/bin:$PATH
  6. Reboot to automatically add JAVA_HOME and J2RE to your PATH.
